# 代码审查（中文）

用中文进行全面的代码审查。

## 何时使用

当用户要求：
- 审查代码质量
- 检查安全性问题
- 优化代码性能
- 改进代码风格

## 审查维度

### 1. 功能性检查
- [ ] 代码是否正确实现了需求？
- [ ] 边界条件是否处理正确？
- [ ] 错误处理是否完善？

### 2. 代码质量
- [ ] 命名是否清晰易懂？
- [ ] 函数/模块是否单一职责？
- [ ] 是否有重复代码可以抽取？
- [ ] 注释是否充分且准确？

### 3. 性能考量
- [ ] 是否存在性能瓶颈？
- [ ] 循环和递归是否合理？
- [ ] 是否有不必要的计算？

### 4. 安全性
- [ ] 是否有注入漏洞？
- [ ] 输入验证是否充分？
- [ ] 敏感数据处理是否安全？

### 5. 可维护性
- [ ] 代码结构是否清晰？
- [ ] 是否便于测试？
- [ ] 文档是否完善？
